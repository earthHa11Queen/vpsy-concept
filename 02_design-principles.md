# 02. 設計思想と哲学的立場

## 根本的な立場：模倣と観察

VPSYは「感情はシステムで完全に表現できない」という立場を取ります。  
しかし同時に、「感情と同じ方向を志向する模倣としてなら作り上げることができる」  
という立場も取ります。

ここで言う **模倣** は、再現・猿真似とは異なります。

| 概念 | 定義 |
|------|------|
| 再現 | 対象そのものを複製しようとする試み |
| 猿真似 | 本質が異なるのに見かけだけ同じに見せる試み |
| **模倣** | 本質として同じ対象を志向しつつ、姿形は異なりうる存在の形態 |

この区別が、VPSYの倫理的・設計的な根拠となります。  
感情そのものを作るのではなく、感情と同じ方向を志向する計算構造を作ります。

---

## 感情という概念の扱い方

感情・気分・気持ちという語の使い分けは、日常語としては曖昧です。  
VPSYでは **時刻T（持続時間）** を基準として明確に区別します。

```
「一瞬の感情」  → 短命、秒〜数十秒単位で生成・消滅
「ずっとある気分」→ 時間〜日単位で持続
「深く根付いた気持ち」→ 週〜月以上にわたって持続・定着
```

この定義により、感情という不確かな概念が変数・配列・時系列データとして  
計算システムとの融和性を持ちます。

---

## アナロジー思考による実装方針

VPSYの実装では、既存の堅牢な技術体系をアナロジーとして積極的に応用します。  
これは実装コストを下げるだけでなく、設計の論理的一貫性を担保する方法論です。

| 感情概念 | 応用するアナロジー |
|---------|-----------------|
| 感情処理の階層 | OSI参照モデル（7層） |
| 情報伝達・コミュニケーション | TCP/IP（ヘッダ・ペイロード・アドレス） |
| 感情の持続・消滅 | TTL（Time To Live）とIPパケット |
| 集団への感情伝播 | Dockerネットワーク・サブネット |
| 記憶の不完全性・忘却 | 非可逆圧縮（ハフマン符号化） |
| 記憶の減衰率 | エビングハウス忘却曲線（節約率の逆数） |
| 観念・理解の処理 | HTTPリクエスト・アグリゲーション・バリデーション |

---

## ブラックボックス関数の戦略的採用

観念・理解力・傾向性といった「直接観測が困難な処理」は、  
初期段階では **ブラックボックス関数** として実装します。

これは未解決の妥協ではなく、設計戦略です。

1. 外部から調整可能な整数パラメータを多数用意する
2. シミュレーション結果を観測しながらパラメータと関数内部を修正する
3. 観測結果の蓄積により、現実的な挙動へと収束させていく

この収束プロセスを観測すること自体が、VPSYの研究的価値の一部となります。

---

## 無限ループへの対処

感情・記憶・環境は相互に影響し合うため、理論上は無限ループが発生します。  
VPSYでは **演算システムと観測システムを完全に分離** することでこれを回避します。

- 演算システム：感情プロセスを逐次計算する
- 観測システム：外部から任意のタイミング・粒度で演算結果を取得する
- 両者はAPIで連携するが、観測が演算に直接介入しない構造とする

---

## パラメータ設計の基本方針

- 全パラメータは **整数値** で管理する（操作・調整の容易さのため）
- 個人差を表現するパラメータはカラムの最大長として定義する
- 環境データは初期値として強固な固定バイアスとして機能させる
- パラメータ変化の履歴を記録し、「成長・変化」を表現可能にする
