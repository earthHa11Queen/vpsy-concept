# 03. 感情処理7層モデル（OSI参照モデル対応）

## 概要

外部刺激が感情として認識されるまでのプロセスを、  
OSI参照モデルに対応した **7層アーキテクチャ** としてモデル化します。

OSI参照モデルを採用する理由は、  
「各層が独立して機能しつつ、上下の層とのみ連携する」という責任分離の原則が、  
感情処理の多層性と論理的に一致するためです。

---

## 7層モデル定義

```
外部刺激（情報入力）
      ↓
┌─────────────────────────────────────────────┐
│ L1. 情報層（物理層）                           │
│     外部からの全ての知覚可能な刺激・情報          │
│     ← 記憶・環境バイアスがここに作用する          │
├─────────────────────────────────────────────┤
│ L2. 観念層（データリンク層）                     │
│     入力情報を個人が持つ概念の枠組みで解釈・抽象化  │
│     複数の入力を一つの解釈へとアグリゲーション      │
├─────────────────────────────────────────────┤
│ L3. 理解層（ネットワーク層）                     │
│     抽象化された情報を論理的に解釈する              │
│     個人の能力差（知性・経験値）による差異が生じる   │
├─────────────────────────────────────────────┤
│ L4. 傾向性層（トランスポート層）                  │
│     個人の性格・無意識の偏りに基づく重み付け        │
│     情報にTTLを付与し、各層で加工・削り出しを行う    │
├─────────────────────────────────────────────┤
│ L5. 感情層（セッション層）                       │
│     瞬間的な感情（喜び・怒りなど）が生成・消滅する   │
│     複数の感情が同時発生し、TTLにより持続・消滅する  │
├─────────────────────────────────────────────┤
│ L6. 気分層（プレゼンテーション層）                │
│     同種の感情が一定時間持続することで気分が形成     │
│     サブネット的に「気分の傾向性」が固まる          │
├─────────────────────────────────────────────┤
│ L7. 気持ち層（アプリケーション層）                │
│     気分がさらに長期的に定着することで気持ちへ昇華   │
│     長期記憶へのインサートが発生する               │
└─────────────────────────────────────────────┘
      ↓
気持ち → 記憶テーブルへ圧縮保存（L1へのフィードバック）
```

---

## 各層の詳細

### L1. 情報層（物理層）

外部から知覚される全ての刺激が情報として入力される層です。

- 五感を通じた知覚情報
- 会話・行動・状況データ
- 記憶テーブルと環境テーブルの参照値がバイアスとして付加される
- 記憶がL1へのインプットとして機能する場合は**L1に含まれる**

### L2. 観念層（データリンク層）

入力された複数の情報を、個人が持つ概念の枠組みで解釈・抽象化する層です。

- 複数ソースからのJSON形式情報を一つの統合JSONへアグリゲーションする処理として実装
- アグリゲーション時の取りこぼし（処理できない情報の欠落）が個性を生む
- **観念モジュールテーブル** からロジックを呼び出し、結果を同レコードに保存する
- ブラックボックス関数の調整ねじとして機能する

処理イメージ（実装アナロジー）：
```
複数のHTTPリクエスト（各情報ソース）
    → 観念アグリゲーター
    → 単一の統合JSON出力
```

### L3. 理解層（ネットワーク層）

観念層で統合されたJSONに対し、バリデーションチェックを行う層です。

- チェックの厳格さ → 固定観念・偏見の表現
- 許容範囲の広さ → 柔軟な理解・認知の偏りのなさの表現
- **理解力パラメータテーブル** で個人差を管理・記録する
- パラメータ変化の経歴が「成長・学習」を表現する

### L4. 傾向性層（トランスポート層）

バリデーション後のデータに係数・重み付けを加える層です。

- 処理の核心：一行配列データへの変換 → 情報量（エントロピー）による昇順並び替え
- 情報量が少ない（よくある情報）→ 感情の振れ幅小
- 情報量が多い（珍しい・衝撃的な情報）→ 感情の振れ幅大
- データに **TTL** を付与し、IPパケットとして扱う
- TTLが単位時間内に枯渇 → 感情として生起せず消滅
- TTLが気分単位時間まで継続 → L6（気分層）へフェーズ移行

### L5. 感情層（セッション層）

傾向性層から渡されたデータをブルチック感情の輪等でマッピングし、  
瞬間的な感情として生成・消滅させる層です。

- **感情定義テーブル** の範囲にマッピングして生起感情を決定
- 複数感情の同時発生を許容
- TTLによる感情の持続・消滅管理
- **感情ステータステーブル** にトランザクションとして記録

### L6. 気分層（プレゼンテーション層）

感情のTTLが気分単位時間まで継続した場合に気分が形成される層です。

- サブネット的に「この単位時間内の感情傾向」が固定される
- 短期記憶テーブルと環境テーブルの影響を受けて制限される
- 気分は短期記憶テーブルへインサートされる
- アブノーマルな情報が連続した場合、気分は不安定化する

### L7. 気持ち層（アプリケーション層）

気分のTTLがさらに継続し定着した状態です。

- 長期記憶テーブルへのインサートが発生する
- 記憶がL1（情報層）へのフィードバックとして作用し始める
- 記憶がL7としての機能を持つ場合は **L7に含まれる**

---

## 記憶・環境の特殊な位置づけ

記憶と環境は特定の単一層に属しません。

| 要素 | L1としての機能 | L7としての機能 |
|------|--------------|--------------|
| 記憶 | 新規情報処理時のバイアスとして入力 | 気持ちが定着した結果として保存 |
| 環境 | 情報解釈時の固定バイアスとして入力 | 長期的な人格・傾向性形成として定着 |
