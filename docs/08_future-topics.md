# 08. 未考察・拡張テーマ

## 概要

現時点で考察が不十分なテーマ、および将来的な拡張として検討中のテーマを記録します。  
これらは「解決すべき問題」として認識しているが、「答えを出すのは留保する」スタンスです。

---

## 拡張スコープ：個人から世界へ

### 集団シミュレーション

個人（Dockerコンテナ）を複数起動し、Dockerネットワークで接続することで  
集団の感情相互作用をシミュレートします。

```
個人コンテナ群 → コミュニティ → 地域 → 国家 → 世界
```

地理的情報データベースや経済格差データを組み合わせることで、  
スラム街と高級住宅街、あるいは異なる文化圏間の感情変化・伝播を観察できます。

**国際間シミュレーションにおけるネットワーク遅延の応用**

| パラメータ | 表現できる現象 |
|-----------|--------------|
| 遅延（Latency） | 情報伝達の物理的・心理的遅れ |
| ジッター（Jitter） | 翻訳・解釈時の意味の変容・誤訳 |
| パケットロス | コミュニケーション断絶・情報欠損 |

## 未解決テーマ

### 音楽・音声情報の処理

音は感情に直接影響を与えますが、現時点でモデル化が不十分です。

**考察の方向性**

- 周波数・帯域・声量をパラメータとして数値化する
- 発声の連続性が意味を変える現象をモデル化する
  - 例：「マジか」が3回連続 → 驚き → 衝撃 → 落胆 と意味が遷移する
- 声のトーンをプロトコルとして定義する（詳細は [05_communication-model.md](05_communication-model.md)）

---

### 外れ値としての「ひらめき・発想」

通常の論理プロセスからは生成されない、偶発的なアイディアや発想をどのようにモデル化するか。

**考察の方向性**

- 傾向性層における確率的な外れ値処理として実装する可能性
- 判断がつかないパラメータ値域を「言い表せない感覚」として扱う
  - これこそが「気持ちがいい」「なんとも言えない」といった表現の計算的定義かもしれない

---

### 死の概念のシステムへの組み込み

「生」と「死」という根源的なテーマをどのようにシステムに実装するか。

**考察の方向性**

- コンテナの停止・削除として表現できる可能性
- 「死の概念を持つこと」が気持ちや傾向性に与える影響のパラメータ化
- 記憶の消去プロセスとの関連性

---

### 集団の意識・社会現象

個体の相互作用から創発する集団心理をどのように観察・測定するか。

**考察の方向性**

- 複数コンテナ間の感情データのアグリゲーション方法
- 「感情の伝染」をパケット転送として表現する
- 社会的証明・集団思考などの心理学理論をパラメータとして組み込む

---
