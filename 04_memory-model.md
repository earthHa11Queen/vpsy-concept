# 04. 記憶モデル

## 概要

VPSYにおける記憶は **静的なデータストア** ではなく、  
時間とともに劣化・圧縮・消去されていく **動的なプロセス** として定義します。

記憶の不完全性・忘却・曖昧さを、非可逆圧縮アルゴリズムで表現します。

---

## 基本方針：非可逆圧縮による記憶の不完全性

記憶は圧縮される際に一部の情報が失われます（非可逆圧縮）。  
復元したときに値がずれ、そのずれた値がさらに記憶される。  
この繰り返しにより、時間とともに記憶の曖昧さが積み重なります。

```
気持ち（原データ）
  → ハフマン符号化（短期記憶）
    → TTL経過・再符号化
      → 長期記憶テーブルへ定着
        → 過去テーブルへ圧縮
          → 昔の記憶テーブルへ再圧縮（ライフステージ単位）
```

---

## ハフマン符号化の応用

ハフマン符号化は「出現頻度が高い要素を短いビット列で表す」アルゴリズムです。  
VPSYでは感情・気分の **生起頻度** に対してこれを適用します。

| 生起頻度 | ビット列長 | 記憶への影響 |
|---------|----------|------------|
| 頻繁に発生した感情・気分 | 短いビット列 | 記憶に残りやすい |
| まれにしか発生しない感情 | 長いビット列 | 記憶容量を圧迫、削除されやすい |

これにより、「強烈で繰り返された感情は残り、些細な感情は淘汰される」  
という記憶の有機的な選択性を表現します。

---

## エビングハウス忘却曲線の応用

忘却曲線の **節約率の逆数** を忘却率として定義します。

```
忘却率 = 1 / 節約率
```

時刻Tの経過に伴い、忘却率に従ってビット列がさらに長くなります（記憶の薄れ）。  
一方、複数回ハフマン符号化されるほど短くなったビット列（強い記憶）は、  
この忘却プロセスに抗いやすくなります。

```
強い記憶（短いビット列）→ 忘却率が適用されても残存しやすい → 長期記憶へ
弱い記憶（長いビット列）→ 忘却率の積が重なる → データとして消去
```

---

## 記憶の4段階と個人差

記憶はテーブルとして4段階で管理されます。

```
短期記憶テーブル
  ↓ （一定時刻T経過後）
長期記憶テーブル
  ↓ （さらなる時刻T経過後）
過去テーブル（長期記憶よりさらに圧縮）
  ↓ （ライフステージ単位での再圧縮）
昔の記憶テーブル（ライフイベント単位でまとめ圧縮）
```

**記憶容量の個人差の表現**

記憶を保持できるデータ容量（カラムの最大長）を個人パラメータとして定義します。  
これにより、記憶力の個人差・記憶の取捨選択の違いを表現できます。

---

## 悪感情の累積と長期影響

短期記憶テーブルに悪感情のレコードが多く蓄積された場合、  
ハフマン符号化によって良い記憶も削除されやすくなり、  
悪感情が長期記憶・過去テーブルにまで波及します。

これにより以下の状況を計算的に表現できます。

- トラウマ（強烈な悪感情の長期定着）
- 環境による固定バイアスの形成
- 「昔の深い傷」に新しい出来事が上書きされにくい状態

---

## 環境データとの関係

環境データは「ある種永続的に続く初期値」として扱います。  
克服プロセスが発生しない限り、環境の影響は固定的なバイアスとして  
L1（情報層）〜L4（傾向性層）全体に作用し続けます。

```
環境データ（初期値・固定バイアス）
  ↓ 継続的に作用
L1〜L4の全処理
  ↓ 結果として
気分・気持ちのサブネット固定化（感情が揺れても気持ちまで波及しにくい）
```

これは、環境要因による感情の鈍化・固定化という現象を  
IPサブネットの固定化というアナロジーで表現したものです。

---

## IPV4との計算的親和性

記憶のビット列構造は、IPv4のビット配列（32bit）との演算的親和性があります。  
具体的には、感情の生起条件を「特定のビット配列がIPv4アドレス長と一致したとき」  
として定義できる可能性を検討中です。

これにより、感情の生起・サブネット化・TTL管理をIPv4の既存演算で  
処理できる可能性があり、実装コストの大幅削減が見込めます。
